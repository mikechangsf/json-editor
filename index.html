Salon JSON-ator

What the heck is this?

This is a web form that allows anyone to create a JSON that matches (mostly) with our Firebase database. It still requires a little bit of validation/post-processing by an engineer. However, this allows generation of a JSON for a new salon, new service for easy entry into our database.
This does not replace the development of the admin project. Only the database schema should serve as a reference, not the UI itself.

To Manually Add a Salon

  1. Add a salon via Salon Entry page by filling out the fields
  2. Once done, get the raw JSON from the right side. Save this as new_salon_SALON_NAME.json
  3. Add the list of Services Entry. Note that in this form,
  4. Like salon entry, get the raw JSON and save it as new_services_SALON_NAME.json
  5. Send both files to an engineer for validation and entry

Differences between genereated JSON and Firebase

  • The list of services is not supposed to be an array. It should be a 'Firebase Array', which is a list of objects with unique keys
  • There's a bug with JSON-editor enumSource and integer. Salon-hours needs to be long (remove quote) and month of holiday needs to be long.
